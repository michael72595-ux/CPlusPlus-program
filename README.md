	Project Summary

Built a C++ clock program that displays time in both 12‑hour and 24‑hour formats.

Solved the problem of giving users a simple, menu‑driven way to adjust and view time.

Reinforced core skills like modular functions, loops, formatting, and user‑friendly design.

	What I Did Well

Broke the program into clean, reusable functions that made the logic easier to follow.

Focused on clear output formatting so the clocks looked organized and professional.

Treated the program like a retail workflow: keep it simple, predictable, and easy for the “customer” (user) to navigate.

	Where I Could Improve

Add stronger input validation to prevent errors and make the program more secure.

Reduce repeated logic in the time‑update functions to make the code more efficient.

Consider turning the program into a Clock class for better scalability and OOP structure.

	Most Challenging Parts

Handling rollover logic (like 23:59:59 → 00:00:00) was tricky at first.

Solved it by breaking the logic into smaller steps and testing each part.

Used debugging, course examples, and documentation as part of my support network — similar to how I troubleshoot issues at work by isolating the problem and checking each step.

	Transferable Skills

Breaking big tasks into manageable pieces.

Writing modular, readable code.

Debugging through small, controlled tests.

Thinking about user experience — something I naturally do from years in retail.

	How I Made the Program Maintainable

Used clear variable and function names.

Added comments that explain why the code works, not just what it does.

Kept each function focused on one job, making future updates easier.

Structured the menu so new features can be added without rewriting everything.
